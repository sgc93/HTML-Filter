Explanation Of The Qestion


Writing an HTML filter that can take an arbitrary HTML document as input and produce a safe HTML document for a user's browser is a difficult task due to the complexity of HTML and the many different kinds of browsers in use. As a result, this project requires a deep understanding of both HTML and browser-specific features.
The goal is to create an HTML filter that takes in any valid or invalid HTML document as input and produces an output that is safe for loading into any browser. The output should preserve all ‚Äúbasic markup‚Äù (fonts, formatting, etc.) while preventing execution of potentially dangerous script. There are two main strategies for achieving this: blocking certain elements or validating tag structure.

What is HTML filter?


HTML Filters are a feature in web programming languages such as PHP, ASP.NET and JSP that provide the ability to sanitize user input and protect an application from malicious code injection. HTML filters examine user-inputted data for specific characters or strings that could be used to execute malicious code and remove them from the data before it is stored in a database or displayed in a page.
